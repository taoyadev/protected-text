---

## 1️⃣ 隐私极客 Alex

### 基本信息

- **年龄**: 28-35岁
- **职业**: 软件工程师 / 安全研究员
- **技术水平**: ⭐⭐⭐⭐⭐
- **占比**: 15-20%

### 使用场景

```
场景1: 存储API密钥和配置
"我需要一个地方快速存储临时的API密钥，
不想用Notion（太慢），不想用本地文件（容易泄露）"

场景2: 团队间分享敏感信息
"需要给同事分享数据库密码，
但不想通过Slack（可能被记录）"

场景3: 跨设备同步代码片段
"在家写的配置，到公司需要用，
但不想push到Git（太零散）"
```

### 核心诉求

- ✅ 强加密（必须是客户端加密）
- ✅ 代码高亮（必须）
- ✅ Markdown支持
- ✅ 版本控制
- ✅ API访问能力
- ⚠️ 会验证加密实现（开源前端）

### 痛点

- ❌ 不信任任何中心化平台
- ❌ 讨厌注册流程
- ❌ 需要快速访问（URL直达）

### 转化策略

1. **技术博客**: "为什么我们使用AES-256-GCM"
2. **开源前端**: GitHub上公开加密代码
3. **技术文档**: 详细的安全架构说明
4. **Pro卖点**: API访问、无限版本历史

### 典型用户故事

> "我是一名DevOps工程师，经常需要在不同环境间复制配置文件。之前用Pastebin但是没加密不安全，用LastPass又太重。Protected Text刚好满足我的需求：快速、加密、支持代码高亮。已升级Pro版，主要为了API集成到我的脚本里。"
> — Alex, DevOps Engineer @ Stripe

## 2️⃣ 隐私保护者 Sarah

### 基本信息

- **年龄**: 25-45岁
- **职业**: 记者 / 律师 / 医疗工作者 / 普通用户
- **技术水平**: ⭐⭐⭐
- **占比**: 35-40%

### 使用场景

```
场景1: 私人日记
"不想用Notion（公司可能看到数据），
不想用本地文件（电脑坏了就没了）"

场景2: 密码和账号管理
"需要记录一些不常用的密码，
1Password太贵，纸质又容易丢"

场景3: 敏感信息临时存储
"需要记录银行信息、医疗记录等，
但只是临时的，不想占用密码管理器"
```

### 核心诉求

- ✅ 简单易用（零学习成本）
- ✅ 可靠加密（但不需要理解技术细节）
- ✅ 跨设备访问
- ✅ 数据不丢失
- ✅ 隐私保证（不被平台看到）

### 痛点

- ❌ 担心数据泄露
- ❌ 不想给太多平台信任
- ❌ 讨厌复杂的界面
- ❌ 担心公司/政府监控

### 转化策略

1. **信任建立**: 首页清晰说明"零知识加密"
2. **简单对比**: "比Notion更私密，比纸质更安全"
3. **社会证明**: 用户评价、媒体报道
4. **Pro卖点**: 数据永久保留、更大存储

### 典型用户故事

> "我是一名记者，经常需要记录采访笔记和线索。之前用的云笔记软件让我不放心，因为不知道平台会不会看我的内容。Protected Text让我感到安心，界面也很简洁。"
> — Sarah, Investigative Journalist

## 3️⃣ 轻量级用户 Mike

### 基本信息

- **年龄**: 20-50岁
- **职业**: 学生 / 自由职业者 / 上班族
- **技术水平**: ⭐⭐
- **占比**: 40-45%

### 使用场景

```
场景1: 跨设备备忘录
"在手机上记录想法，
回家在电脑上整理"

场景2: 与朋友分享信息
"和室友分享WiFi密码，
但不想通过微信（会被存档）"

场景3: 临时记录
"会议记录、购物清单、待办事项，
不想为了这些小事用Notion"
```

### 核心诉求

- ✅ 快速（打开就能用）
- ✅ 简单（无需注册）
- ✅ 免费（基础功能）
- ✅ 够用就好（不需要太多功能）

### 痛点

- ❌ Notion太重，启动慢
- ❌ 不想为简单需求注册账号
- ❌ 不需要复杂功能
- ❌ 手机端体验差

### 转化策略

1. **极简落地页**: 3秒内理解产品
2. **零门槛**: 无需注册即可使用
3. **移动优化**: PWA + 响应式设计
4. **Pro卖点**: 可能不转化（OK，免费用户也有价值）

### 典型用户故事

> "我只是需要一个地方快速记录东西，不想注册、不想安装app。Protected Text完美满足我的需求，打开网址就能用。"
> — Mike, College Student

## 5️⃣ 误导流量用户 Tom

### 基本信息

- **年龄**: 任意
- **搜索意图**: 复制受保护的PDF/网站内容
- **技术水平**: ⭐
- **占比**: 30-35% （非目标用户）

### 搜索关键词

```
- "copy text from protected pdf"
- "how to copy text from protected website"
- "extract text from protected pdf"
```

### 核心需求

- 想要**绕过**内容保护
- 不是想要**创建**保护内容

### 转化策略

⚠️ **这不是我们的目标用户，但可以部分转化**

#### 方案1: 教育内容

创建博客文章:

```
标题: "为什么你不应该复制受保护的内容"
内容: 教育版权意识 → 引导到我们的加密笔记产品
```

#### 方案2: 替代产品

```
落地页: "看起来你在找PDF复制工具？
       你可能真正需要的是保护你自己的内容！"
CTA: "免费创建你的加密笔记 →"
```

#### 方案3: SEO内容

```
博客: "如何保护你的PDF不被复制"
内容: 讲解内容保护 → 推广我们的文本加密功能
```

### 转化率预期

- 📊 预计转化率: 0.5-1%
- 📊 SEO价值: 高（大量搜索量）
- 📊 品牌风险: 低（只要明确区分功能）

## 📝 用户访谈问题清单

### 在产品发布前，应该问用户：

#### 关于隐私

1. 你在哪里存储敏感信息？为什么？
2. 你最担心的隐私问题是什么？
3. 你如何验证一个产品是否安全？

#### 关于使用习惯

4. 你多久需要跨设备访问笔记？
5. 你通常记录什么类型的内容？
6. 你愿意为隐私付费吗？多少钱合理？

#### 关于竞品

7. 你用过类似的产品吗？体验如何？
8. 什么功能是你必须有的？
9. 什么功能让你愿意付费？

#### 关于产品

10. 你会推荐给朋友吗？为什么？
11. 如果免费版限制存储大小，你能接受吗？
12. 你需要团队协作功能吗？

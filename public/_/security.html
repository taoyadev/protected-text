Security

CloudNote D1 uses client-side encryption. The server stores only ciphertext and cannot decrypt without your password.

Crypto design

  • Key derivation: PBKDF2 (SHA-256), 100,000 iterations
  • Encryption: AES-256-GCM
  • Per-note salt: random 16 bytes (unique per slug)
  • Per-save IV: random 12 bytes

Threat model

  • If the database is leaked, attackers get ciphertext and may attempt offline password guessing.
  • Choose a strong password. This is the primary defense.
  • Slug secrecy helps, but should not be your only defense.

Operational recommendations

  • Enable Cloudflare dashboard Rate Limiting rules to reduce abusive writes.
  • Back up D1 regularly (export).
